## 一、首次使用

### 1.1 登录系统

启动 Toonflow 后，使用默认账户登录：

| 项目 | 值 |
|------|-----|
| 账号 | `admin` |
| 密码 | `admin123` |

> **重要**：登录后请立即修改默认密码。

### 1.2 配置 AI 模型

Toonflow 需要配置外部 AI 服务才能使用各项生成功能。进入 **设置 > AI 模型管理**，添加至少以下三类模型：

#### 文本模型（必需）

用于大纲生成、剧本生成、提示词润色等所有文本类 AI 任务。

| 字段 | 说明 | 示例 |
|------|------|------|
| 类型 | 选择 `text` | text |
| 模型名称 | 从支持列表中选择 | deepseek-chat |
| 接口地址 | API Base URL | https://api.deepseek.com |
| API Key | 服务商提供的密钥 | sk-xxx |

**支持的文本模型**（部分列举）：
- DeepSeek：deepseek-chat、deepseek-reasoner
- 通义千问：qwen-plus、qwen-max、qwen-turbo
- 智谱 GLM：glm-4-plus、glm-4-flash
- 豆包：doubao-pro-256k、doubao-lite-128k
- OpenAI：gpt-4o、gpt-4o-mini、o1、o3-mini
- Claude：claude-sonnet-4-20250514
- Gemini：gemini-2.5-pro、gemini-2.5-flash
- Grok：grok-3、grok-3-mini

#### 图片模型（必需）

用于角色四视图、场景参考图、道具图、分镜图生成。

| 字段 | 说明 | 示例 |
|------|------|------|
| 类型 | 选择 `image` | image |
| 模型名称 | 从支持列表中选择 | doubao-seedream-3-0 |
| 接口地址 | API Base URL | — |
| API Key | 服务商提供的密钥 | — |

**支持的图片模型**：
- 豆包 SeedDream 系列
- 可灵（Kling）
- Gemini（imagen 系列）
- Vidu
- RunningHub（ComfyUI 工作流）
- 其他 OpenAI 兼容接口

#### 视频模型（必需）

用于将分镜图生成视频片段。

| 字段 | 说明 | 示例 |
|------|------|------|
| 类型 | 选择 `video` | video |
| 厂商 | 视频服务商 | volcengine |
| 模型名称 | 具体模型 | doubao-seedance-2-0-260128 |
| 接口地址 | API Base URL | — |
| API Key | 服务商提供的密钥 | — |

**支持的视频模型**：

| 厂商 | 模型 | 时长 | 分辨率 | 特色 |
|------|------|------|--------|------|
| Vidu | viduq3-pro | 1-16秒 | 540p/720p/1080p | 支持音频生成 |
| Vidu | viduq2-pro | 1-10秒 | 540p/720p/1080p | 首尾帧、参考图、文本 |
| 可灵 | kling-v2-6(PRO) | 5/10秒 | 1080p | 文本、首尾帧 |
| 火山引擎 | doubao-seedance-2-0 | 4-15秒 | 480p/720p | 多图、音频、尾帧可选 |
| Wan | wan2.1-t2v/i2v | 多种时长 | 多种分辨率 | 文本转视频、图转视频 |
| OpenAI | sora | 多种时长 | 多种分辨率 | 网格图拼接模式 |
| Gemini | veo | 多种时长 | 多种分辨率 | 文本和图片输入 |

### 1.3 分配模型用途

添加模型后，需要在 **设置 > 模型用途分配** 中将模型指派给具体的功能模块：

| 用途 | 模型类型 | 说明 |
|------|----------|------|
| 分镜 Agent | 文本 | 分镜对话的主 AI |
| 分镜 Agent 图片生成 | 图片 | 分镜图绘制 |
| 大纲故事线 Agent | 文本 | 大纲对话的主 AI |
| 资产提示词润色 | 文本 | 角色/场景/道具描述优化 |
| 资产图片生成 | 图片 | 角色/场景/道具图绘制 |
| 剧本生成 | 文本 | 从大纲生成剧本 |
| 视频提示词生成 | 文本 | 分镜转视频提示词 |
| 图片编辑 | 图片 | 分镜图局部修改 |

> **提示**：建议文本类功能使用支持 Tool Calling 的模型（如 deepseek-chat、gpt-4o、claude-sonnet），Agent 功能需要此能力。

### 1.4 修改账户信息

进入 **设置 > 用户管理**，修改用户名和密码。

## 二、创作流程总览

Toonflow 的完整创作流程：

```
小说导入 → 大纲生成 → 资产管理 → 剧本生成 → 分镜制作 → 视频生成
```

详细步骤：

```
1. 创建项目（设定名称、类型、画风）
       ↓
2. 导入小说（按卷/章节结构）
       ↓
3. AI 生成故事线（概括核心情节）
       ↓
4. AI 生成分集大纲（角色、场景、道具、情节节点）
       ↓
5. 管理资产（角色/场景/道具 → 提示词润色 → AI 生图）
       ↓
6. AI 生成剧本（基于大纲+原文）
       ↓
7. AI 生成分镜（拆片段 → 生成镜头 → AI 生图）
       ↓
8. AI 生成视频（分镜图+提示词 → 视频片段）
```

> 每个步骤都可独立操作、反复调整，不必严格线性执行。

## 三、创建项目

1. 点击 **新建项目**
2. 填写项目信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| 项目名称 | 作品名 | 仙侠奇缘 |
| 项目类型 | 内容分类 | 短剧 / 漫剧 |
| 简介 | 故事背景概述 | 讲述一位少年修仙的故事... |
| 小说类型 | 题材分类 | 仙侠 / 都市 / 悬疑 / 科幻 |
| 画风 | AI 绘图风格 | 从 100+ 预设画风中选择 |
| 视频比例 | 输出视频比例 | 16:9（横屏）/ 9:16（竖屏） |

3. 点击 **确认创建**

> **画风选择**：系统内置 100+ 种画风，涵盖 2D 动画、写实、3D、赛璐珞、漫画、国画、油画等多种风格。每种画风附带中英文提示词和预览图，选择后会自动应用到所有 AI 图片生成。

---

## 四、导入小说

1. 进入项目后，切换到 **小说** 页面
2. 按章节导入小说内容：
   - **卷**：小说的大分类（如：第一卷）
   - **章节标题**：章节名称（如：第一章 少年出山）
   - **章节内容**：正文文本
3. 支持批量导入多个章节

> **提示**：章节划分会影响后续大纲生成的粒度。建议按原著章节结构导入，每章 2000-5000 字为佳。

## 五、生成大纲与故事线

大纲生成由 AI Agent 系统完成，通过实时对话交互，逐步从小说中提炼出结构化的创作蓝图。

### 5.1 与大纲 Agent 对话

进入项目的 **大纲** 页面，系统会启动一个 WebSocket 实时对话窗口。Agent 系统包含三个角色：

| 角色 | 职责 |
|------|------|
| **主协调** | 管理整体流程，协调其他 Agent |
| **故事师（AI1）** | 分析原著，生成故事线 |
| **大纲师（AI2）** | 将故事线细化为分集大纲 |
| **导演** | 审查故事线和大纲质量，提出修改建议 |

### 5.2 故事线生成

1. 在对话中告诉 Agent 你想创作的方向（如："请根据第1-3章生成故事线"）
2. AI1（故事师）会分析原著文本，生成故事线概要
3. 故事线包含：核心冲突、人物关系、情节走向
4. 你可以要求修改或重新生成

### 5.3 大纲生成

1. 确认故事线后，告诉 Agent 你需要的目标集数和每集时长
2. AI2（大纲师）会生成结构化的分集大纲，包含：

| 字段 | 说明 |
|------|------|
| 角色列表 | 本集出场角色及其描述 |
| 道具列表 | 本集出现的关键道具 |
| 场景列表 | 本集涉及的场景 |
| 剧情主干 | 起承转合四阶段的详细描述 |
| 情感曲线 | 各节点的情绪标记 |
| 经典台词 | 高潮段落的标志性台词 |
| 视觉亮点 | 值得重点展现的画面 |
| 章节范围 | 对应原著的章节 |

### 5.4 导演审查

1. 大纲生成后，导演 Agent 会自动审查并给出专业建议
2. 审查维度：节奏、冲突强度、人物弧光、视觉可行性
3. **导演建议仅供参考**，最终决定权在你手中
4. 你可以根据建议让 AI2 修改，或直接确认通过

> **对话技巧**：
> - 可以随时说"修改第X集的XXX"来调整特定内容
> - 说"删除第X集大纲"会触发两步确认流程
> - 对话历史会自动保存，下次进入可继续

## 六、管理资产

资产是创作的视觉基础，包括角色、场景和道具。大纲确认后，系统会自动提取资产列表，你也可以手动添加。

### 6.1 角色资产

每个角色包含：
- **名称**：角色名
- **描述**：外貌、性格、身份等详细设定
- **提示词**：AI 润色后的绘图提示词
- **参考图**：AI 生成的角色四视图（正面/侧面/背面/细节）

### 6.2 场景资产

每个场景包含：
- **名称**：场景名
- **描述**：环境、氛围、光照等
- **提示词**：AI 润色后的绘图提示词
- **参考图**：AI 生成的场景参考图（全景+三视角）

### 6.3 道具资产

每个道具包含：
- **名称**：道具名
- **描述**：外观、材质、用途等
- **提示词**：AI 润色后的绘图提示词
- **参考图**：AI 生成的道具图

### 6.4 AI 提示词润色

1. 选择一个资产，点击 **AI 润色提示词**
2. 系统会结合以下信息自动优化提示词：
   - 原著相关章节文本
   - 项目画风设定
   - 小说类型和背景
   - 资产的名称和描述
3. 润色后的提示词会比手写更适合 AI 绘图

### 6.5 AI 图片生成

1. 确认提示词后，点击 **生成图片**
2. 可选择是否上传参考图（作为风格参考）
3. AI 会根据提示词和画风生成对应图片：
   - **角色**：2x2 网格四视图（正面全身、右侧面、背面、特征细节）
   - **场景**：左右分栏参考图（40% 全景 + 60% 三视角）
   - **道具**：单张道具特写

> **提示**：如果生成效果不满意，可以修改提示词后重新生成，不会覆盖已有图片。

## 七、生成剧本

1. 进入项目的 **剧本** 页面
2. 选择要生成剧本的大纲（集）
3. 点击 **生成剧本**
4. AI 会基于以下内容自动创作：
   - 结构化大纲（剧情主干、角色、场景、道具）
   - 对应章节的原著文本
   - 情感曲线和经典台词

**剧本特点**：
- 严格按大纲的起承转合顺序展开
- 包含场景描述、角色动作、对白
- 经典台词作为角色对话出现在高潮段落
- 视觉亮点按大纲顺序全部呈现
- 每段描述包含场景名称和可视化动作，便于后续分镜拆分
- 以【黑屏】结尾，用于视频片尾过渡
- 动态字数：根据大纲复杂度自动调整（500字起）

> **编辑**：生成后可以手动编辑剧本内容，修改对白、调整节奏。

## 八、制作分镜

分镜是将剧本转化为可视画面的关键步骤，由分镜 Agent 系统完成。

### 8.1 与分镜 Agent 对话

进入 **分镜** 页面，系统启动分镜 Agent 对话窗口。Agent 会自动加载：
- 当前剧本内容
- 项目资产列表（角色/场景/道具的名称和描述）

### 8.2 片段拆分

1. 告诉 Agent 需要对剧本进行分镜
2. **片段师** 会将剧本拆分为多个叙事片段
3. 每个片段对应一个连贯的场景/情绪单元

### 8.3 分镜生成

1. 对每个片段，**分镜师** 生成具体的镜头描述
2. 每个片段默认生成 4 个镜头（可自定义数量）
3. 每个镜头包含：

| 要素 | 说明 |
|------|------|
| 景别 | 远景/全景/中景/近景/特写/大特写 |
| 机位 | 平视/俯拍/仰拍/过肩/主观视角 |
| 构图 | 三分法/中心/对角线/框架构图 |
| 光线 | 光源方向、质感（硬光/柔光）、色温 |
| 人物 | 位置、姿态、表情、动作 |
| 场景 | 环境细节、前景背景 |
| 氛围 | 情绪关键词 |

**对话场景处理**：
- 对话场景只描述说话者的表情、口型、肢体语言
- 不在画面提示词中包含台词文字
- 采用正反打（过肩镜头交替）技法

### 8.4 分镜图片生成

分镜确认后，系统会自动生成分镜图片：

1. 镜头描述经 **提示词优化师** 转换为英文绘图 prompt
2. 按网格布局排列（16:9 = 3列，9:16 = 2列）
3. AI 批量生成网格图后自动切割为单张
4. 每张图对应一个镜头

**提示词优化规则**：
- 人物名、场景名、道具名保持原文（不翻译）
- 每个 prompt 控制在 25-40 个英文单词
- 自动追加画质后缀和画风标签
- 支持"纯黑图"作为过渡帧

### 8.5 手动编辑分镜

- **替换镜头**：上传自定义图片替换 AI 生成的图
- **编辑提示词**：修改后重新生成单个镜头
- **图片编辑**：对已有分镜图进行局部修改
- **超分辨率**：批量提升分镜图清晰度

## 九、生成视频

### 9.1 视频生成模式

根据所选视频模型，支持以下生成模式：

| 模式 | 输入 | 说明 |
|------|------|------|
| **文本模式** (text) | 纯文字描述 | 根据文字描述生成视频，无参考图 |
| **单图模式** (single) | 1张图片 + 文字 | 以单张分镜图为起点生成运动视频 |
| **多图模式** (multi) | 多张图片 + 文字 | 多张图片拼接为网格输入（适用于 Sora 等模型） |
| **首尾帧模式** (startEnd) | 起始帧 + 结束帧 | 指定起始和结束画面，AI 补全中间过渡 |

### 9.2 配置视频参数

| 参数 | 说明 | 可选值 |
|------|------|--------|
| 分辨率 | 输出视频分辨率 | 480p / 540p / 720p / 1080p |
| 时长 | 视频片段长度 | 1-16秒（因模型而异） |
| 画面比例 | 视频宽高比 | 16:9 / 9:16 / 1:1 / 4:3 / 3:4 / 21:9 |
| 音频 | 是否生成配音 | 开启/关闭（部分模型支持） |
| 提示词 | 运动/动作描述 | 描述画面中的动态变化 |

### 9.3 批量生成

1. 进入视频页面，选择需要生成视频的分镜
2. 系统自动关联分镜图和提示词
3. 点击 **批量生成** 启动异步处理
4. 视频生成为后台任务，可在 **任务列表** 中查看进度：
   - 状态 0：生成中
   - 状态 1：成功
   - 状态 -1：失败（可查看失败原因）

### 9.4 视频提示词生成

每个分镜可自动生成 Motion Prompt（视频动作提示词）：

1. 选择分镜，点击 **生成视频提示词**
2. AI 分析分镜图 + 分镜描述 + 剧本上下文
3. 输出结构化的 Motion Prompt：

```json
{
  "time": 5,
  "name": "少年转身凝视",
  "content": "中景。镜头缓慢推进，角色身体微微紧绷，神情凝重，缓缓转头注视门口..."
}
```

**Motion Prompt 规则**：
- 首句必须是画面类型（前景/近景/中景/远景/全景）
- 只描述动态变化，不重复静态画面元素
- 字数根据时长动态调整（每秒约 20 字）
- 包含：镜头运动、角色动作、环境动态、速度节奏

## 十、系统设置

### 10.1 AI 模型管理

**设置 > AI 模型管理**

- **添加模型**：填写类型、名称、接口地址、API Key
- **编辑模型**：修改已有模型的配置
- **删除模型**：移除不需要的模型
- **测试模型**：验证 AI/图片/视频模型连通性

> 每个模型需要指定 `manufacturer`（厂商标识），系统据此选择对应的 API 适配器。

将已添加的模型分配给 8 个功能模块（见 [1.3 节](#13-分配模型用途)）。

> **注意**：Agent 功能（大纲 Agent、分镜 Agent）需要使用支持 Tool Calling 的文本模型，否则工具调用会失败。

## 附录：常见问题

### Q: Agent 对话没有响应？
**A**: 检查以下几点：
1. 确认已在 **模型用途分配** 中为对应 Agent 指定了模型
2. 确认所选模型支持 Tool Calling（deepseek-reasoner、glm-z1-flash 等推理模型不支持）
3. 检查 API Key 是否有效（可在设置中使用"测试"功能验证）

### Q: 图片生成失败？
**A**:
1. 确认已配置图片模型并分配给对应用途
2. 检查图片模型的 API Key 和接口地址
3. 部分模型对提示词内容有审核限制，可尝试调整描述

### Q: 视频生成一直显示"生成中"？
**A**:
1. 视频生成为异步任务，通常需要 1-5 分钟
2. 如果长时间无变化，检查视频模型配置
3. 状态为 -1 表示失败，可查看错误原因

### Q: 如何更换画风？
**A**: 画风在 **创建项目** 时选择。已创建的项目可在项目设置中修改画风，但已生成的图片不会自动更新，需要重新生成。

### Q: 提示词自定义后效果变差？
**A**: 清空自定义值即可恢复系统默认提示词。建议在默认提示词基础上微调，而非完全重写。

### Q: 如何备份数据？
**A**:
- **桌面版**：数据存储在用户数据目录（`userData/uploads` 和 SQLite 数据库）
- **Docker 部署**：配置 volumes 持久化 `uploads` 和数据库目录
- **服务器部署**：备份项目目录下的 `uploads` 文件夹和 SQLite 数据库文件

### Q: 支持多用户吗？
**A**: 当前版本为单用户设计，所有数据共享同一账户。多用户支持在规划中。

### Q: API 调用费用？
**A**: Toonflow 本身免费开源，但 AI 模型的 API 调用会产生费用，具体取决于你使用的服务商和模型。建议：
- 调试阶段使用低成本模型（如 deepseek-chat、doubao-lite）
- 正式生产使用高质量模型（如 gpt-4o、qwen-max）
